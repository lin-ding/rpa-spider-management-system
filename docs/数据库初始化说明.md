# 数据库初始化脚本说明

## 执行方式

### 方式一：使用 MySQL 命令行工具

```bash
# 1. 登录 MySQL
mysql -u root -p

# 2. 执行 SQL 脚本
source D:/spring金融/rpa-spider-management-system/src/main/resources/db/schema.sql
```

### 方式二：使用 Navicat/MySQL Workbench 等图形化工具

1. 打开 Navicat 或 MySQL Workbench
2. 连接到 MySQL 数据库
3. 新建查询
4. 打开 `schema.sql` 文件
5. 执行全部 SQL 语句

### 方式三：使用 IDEA 数据库工具

1. 在 IDEA 中打开 Database 面板
2. 右键点击数据源 -> Run Data Source Script
3. 选择 `schema.sql` 文件
4. 执行

## 常见问题

### Q1: 执行时提示数据库不存在？
A: `schema.sql` 中已包含 `CREATE DATABASE IF NOT EXISTS rpa_system` 语句，如果仍报错，可手动创建数据库：
```sql
CREATE DATABASE rpa_system DEFAULT CHARACTER SET utf8mb4;
```

### Q2: 字符集问题？
A: 确保 MySQL 服务器支持 `utf8mb4` 字符集，可在 `my.cnf` 或 `my.ini` 中添加：
```ini
[mysqld]
character-set-server=utf8mb4
collation-server=utf8mb4_general_ci
```

### Q3: 表已存在错误？
A: SQL 脚本中包含 `DROP TABLE IF EXISTS` 语句，会先删除已存在的表。如果是生产环境，请谨慎执行。

### Q4: 外键约束错误？
A: 脚本按照正确的顺序创建表（先主表后关联表），如果出现外键错误，检查是否有残留数据。

## 注意事项

1. **开发环境**: 脚本会在每次执行前删除已有表，请勿在生产环境直接执行
2. **密码加密**: 默认密码使用 MD5 加密，实际项目建议使用 BCrypt
3. **逻辑删除**: 系统使用逻辑删除（deleted 字段），物理删除需谨慎
4. **自增 ID**: 使用雪花算法生成 ID（ASSIGN_ID），非自增

## 重置数据库

如需重置数据库到初始状态，重新执行 `schema.sql` 即可。
