# 快速开始指南

## 一、环境准备

### 必需环境
1. **JDK 17+**
   ```bash
   java -version
   ```

2. **Maven 3.6+**
   ```bash
   mvn -version
   ```

3. **MySQL 5.7+**
   ```bash
   mysql --version
   ```

4. **Node.js 14+** (前端开发用)
   ```bash
   node -v
   ```

## 二、项目搭建步骤

### Step 1: 克隆/下载项目
```bash
cd D:/spring金融/rpa-spider-management-system
```

### Step 2: 初始化数据库
```bash
# 方式一：命令行
mysql -u root -p < src/main/resources/db/schema.sql

# 方式二：使用图形化工具打开 src/main/resources/db/schema.sql 并执行
```

### Step 3: 修改数据库配置
编辑 `src/main/resources/application.properties`:
```properties
spring.datasource.url=jdbc:mysql://localhost:3306/rpa_system?useUnicode=true&characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai
spring.datasource.username=root
spring.datasource.password=你的密码
```

### Step 4: 编译项目
```bash
mvn clean install
```

### Step 5: 启动项目
```bash
# 方式一：使用 Maven
mvn spring-boot:run

# 方式二：运行主类
# 在 IDEA 中右键点击 RpaDemoApplication.java -> Run
```

启动成功后显示：
```
====================================
RPA 爬虫管理系统启动成功！
接口地址：http://localhost:8080/api
====================================
```

### Step 6: 测试接口
使用 Postman 或 Apifox 导入 `docs/API 测试集合.postman_collection.json`

测试登录接口：
- URL: `http://localhost:8080/api/auth/login`
- Method: POST
- Body:
```json
{
  "username": "admin",
  "password": "admin123"
}
```

## 五、技术要点

### 1. 密码加密
当前使用 MD5 加密：
```java
String encryptedPassword = SecureUtil.md5(password);
```

生产环境建议使用 BCrypt。

### 2. JWT Token
使用 Hutool 的 JWT 实现：
- 有效期：24 小时
- 载荷包含：userId, username
- 请求头格式：`Authorization: Bearer {token}`

### 3. MyBatis-Plus
- 使用分页插件
- 自动填充字段（createTime, updateTime）
- 逻辑删除（deleted 字段）
- 雪花算法生成 ID

### 4. 统一返回格式
所有接口返回 `Result<T>` 对象：
```java
{
  "code": 200,
  "message": "操作成功",
  "data": {},
  "timestamp": 1234567890
}
```

## 六、常见问题排查

### Q1: 启动时报"数据库连接失败"
**解决方案**:
1. 检查 MySQL 服务是否启动
2. 确认数据库用户名密码正确
3. 确认数据库 `rpa_system` 已创建

### Q2: 登录提示"用户名或密码错误"
**解决方案**:
1. 确认 SQL 脚本已正确执行
2. 默认密码是 `admin123`
3. 检查数据库中密码是否为 MD5 加密

### Q3: Maven 依赖下载失败
**解决方案**:
```bash
# 清理本地仓库缓存后重新下载
mvn dependency:purge-local-repository
mvn clean install
```

### Q4: 端口被占用
**解决方案**:
编辑 `application.properties` 修改端口：
```properties
server.port=8081
```

## 七、开发建议

### 1. 代码规范
- 遵循阿里巴巴 Java 开发手册
- 使用 Lombok 简化代码
- Controller 层只做参数接收和结果返回
- Service 层处理业务逻辑
- 异常统一由 GlobalExceptionHandler 处理

### 2. Git 提交规范
```bash
# 功能开发
git commit -m "feat: 添加用户管理功能"

# Bug 修复
git commit -m "fix: 修复登录验证问题"

# 文档更新
git commit -m "docs: 更新 README"

# 代码重构
git commit -m "refactor: 重构用户服务层"
```

### 3. 分支管理
- `main`: 主分支，稳定版本
- `dev`: 开发分支
- `feature/*`: 功能分支（如 feature/user-management）
