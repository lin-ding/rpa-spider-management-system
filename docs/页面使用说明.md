# 页面使用说明

## 1. 文档目的
本文档用于帮助测试人员、实施人员或普通使用人员快速理解当前系统页面的用途和基本使用方式。  
当前系统主要包括以下模块：

- 流程管理
- 机器人管理
- 任务管理
- 执行监控

## 2. 系统整体使用顺序
系统当前推荐的使用顺序如下：

1. 在流程管理中创建流程并完成流程设计
2. 在机器人管理中创建机器人并确认机器人状态可用
3. 在任务管理中创建任务，将流程和机器人绑定到任务上
4. 从任务管理中启动任务执行
5. 在任务详情页或执行监控页查看执行结果

说明：

- 流程管理用于“定义和调试流程”
- 机器人管理用于“维护执行资源”
- 任务管理用于“正式发起业务执行”
- 执行监控用于“查看全局执行结果”

## 3. 流程管理使用说明

### 3.1 页面用途
流程管理用于维护流程模板。  
一个流程通常由多个阶段组成，每个阶段可以配置脚本。

### 3.2 流程列表页
流程列表页主要用于：

- 查看已有流程
- 查询流程名称
- 按状态筛选流程
- 新建流程
- 编辑流程
- 进入流程设计页
- 调试流程
- 删除流程

### 3.3 创建流程
在“创建流程”页面中需要填写：

- 流程编码
- 流程名称
- 描述
- 状态

说明：

- 流程编码建议保持唯一
- 流程状态建议在设计完成后再启用

### 3.4 编辑流程
编辑流程用于修改流程的基础信息，例如：

- 流程名称
- 描述
- 状态

说明：

- 编辑页与创建页共用同一页面
- 打开编辑页时会自动加载流程详情

### 3.5 流程设计页
流程设计页用于维护流程的阶段信息。  
在流程设计页中可以：

- 新增阶段
- 删除阶段
- 编辑阶段名称
- 调整阶段顺序
- 编写阶段脚本
- 保存流程设计
- 进行语法检查
- 调试整个流程

### 3.6 阶段脚本语言
当前阶段脚本支持：

- Groovy
- Python

说明：

- 新增阶段时默认脚本语言为 `Groovy`
- 老流程没有脚本语言字段时，会默认按 `Groovy` 处理

### 3.7 语法检查
流程设计页中的“语法检查”用于检查当前阶段脚本是否可通过检查。

说明：

- Groovy：执行真实语法检查
- Python：执行真实语法检查
- 若检查失败，会弹出结果窗口并显示错误信息

### 3.8 调试流程
流程设计页中的“调试流程”用于调试当前流程定义。

说明：

- 该功能仅用于开发调试
- 调试结果会弹出窗口展示
- 结果内容为 JSON 格式
- 正式业务执行请在任务管理中发起

## 4. 机器人管理使用说明

### 4.1 页面用途
机器人管理用于维护系统中的执行机器人。

### 4.2 机器人列表页
机器人列表页主要用于：

- 查看机器人列表
- 查询机器人名称或编码
- 按状态筛选
- 查看机器人状态统计
- 新建机器人
- 编辑机器人
- 查看机器人详情
- 手动上报心跳
- 修改机器人状态
- 删除机器人

### 4.3 创建机器人
创建机器人时填写机器人基础信息即可。

说明：

- 创建后机器人应处于可用初始状态
- 若机器人长时间未上报心跳，系统可能自动将其判定为离线

### 4.4 机器人状态
当前常见状态包括：

- 禁用
- 空闲
- 工作中
- 离线
- 异常

说明：

- 空闲机器人可用于绑定任务并执行
- 工作中机器人表示当前正在执行任务
- 离线或异常机器人不建议用于任务执行

### 4.5 心跳上报
心跳用于刷新机器人的在线状态。  
当机器人状态异常或离线时，可通过心跳上报刷新状态。

## 5. 任务管理使用说明

### 5.1 页面用途
任务管理是系统中的正式执行入口。  
任务的核心作用是：

- 绑定流程
- 绑定机器人
- 填写业务参数
- 发起正式执行

### 5.2 任务列表页
任务列表页主要用于：

- 查看任务列表
- 查询任务编码或任务名称
- 按当前状态筛选
- 按流程筛选
- 按机器人筛选
- 创建任务
- 启动任务
- 取消排队任务
- 重试失败任务
- 查看任务详情
- 编辑任务
- 删除任务

### 5.3 创建任务
创建任务时需要填写的核心信息包括：

- 任务编码
- 任务名称
- 关联流程
- 关联机器人
- 企业名称
- 纳税人识别号
- 分类
- 优先级
- 备注

说明：

- 任务本身不等于流程
- 任务是“流程 + 机器人 + 业务参数”的绑定对象

### 5.4 启动任务
点击“启动”后，系统会根据任务绑定的流程和机器人发起执行。

说明：

- 如果机器人空闲，任务会直接执行
- 如果机器人忙碌，任务会进入排队
- 同一个机器人下的任务是串行执行的
- 不同机器人上的任务可以并行执行

### 5.5 取消任务
当前“取消”主要用于取消待执行或排队中的任务。

说明：

- 执行中的任务当前不支持直接取消

### 5.6 重试任务
当任务最近一次执行失败时，可以使用“重试”再次发起执行。

说明：

- 重试会创建新的执行记录
- 原有执行历史会保留

### 5.7 任务详情页
任务详情页用于查看任务完整信息，包括：

- 基本信息
- 当前状态
- 最近结果
- 执行历史
- 阶段日志

说明：

- 当前状态用于表示任务当前是否处于待执行、排队中、执行中
- 最近结果用于表示最近一次执行是否成功或失败

## 6. 执行监控使用说明

### 6.1 页面用途
执行监控用于查看全局范围内的执行记录。

### 6.2 页面功能
执行监控页支持：

- 按任务名称查询
- 按机器人名称查询
- 按状态筛选
- 按时间范围筛选
- 查看执行详情
- 查看阶段执行结果
- 分页查看执行记录

### 6.3 执行详情
执行详情窗口会展示：

- 执行状态
- 开始时间
- 结束时间
- 耗时
- 错误信息
- 阶段执行记录

## 7. 常见使用场景

### 场景 1：新建并执行一个任务

1. 进入流程管理，创建流程并完成设计
2. 进入机器人管理，创建机器人并确认机器人可用
3. 进入任务管理，创建任务并绑定流程和机器人
4. 在任务列表中点击“启动”
5. 在任务详情页查看执行结果

### 场景 2：调试流程脚本

1. 进入流程管理
2. 打开某个流程的设计页
3. 选择阶段并编写脚本
4. 点击“语法检查”
5. 如需验证整体流程，再点击“调试流程”

### 场景 3：查看任务执行情况

1. 进入任务管理
2. 打开任务详情
3. 查看执行历史
4. 查看阶段日志
5. 如需全局查看，可进入执行监控页

## 8. 当前使用注意事项

- 流程管理中的“调试流程”仅用于调试，不是正式执行入口
- 正式业务执行请统一从任务管理发起
- 阶段脚本当前支持 Groovy 和 Python
- Java 当前不作为阶段脚本语言开放
- 单个机器人内部任务串行执行
- 多个机器人之间可并行执行任务

## 9. 页面入口简要说明

- 首页  
  查看整体概览信息

- 流程管理 / 流程列表  
  创建、编辑、设计和调试流程

- 机器人管理 / 机器人列表  
  创建、维护和查看机器人状态

- 任务管理 / 任务列表  
  创建任务并正式发起执行

- 任务管理 / 执行监控  
  查看全局执行记录和执行详情
